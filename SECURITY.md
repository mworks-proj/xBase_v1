# Security Policy

## Reporting a Vulnerability
- Please report security issues privately via **GitHub Security Advisories** (preferred) or email `security@xmerch.dev`.
- Include reproduction steps, affected endpoints/flows, and any logs you can safely share. Avoid attaching secrets or production data.

## Response Expectations
- We aim to acknowledge reports within **2 business days**.
- We will coordinate on fixes, request clarifications or PoCs as needed, and publish a patch and disclosure timeline once resolved.

## Scope
- This policy covers the xBase public repo (Next.js app, Supabase Edge Functions, migrations, and deployment scripts).
- Do not test against production systems you do not own; use local or test environments.
- Please avoid DoS or spam-style testing.
